⚠️ Оценка рисков и план митигации

🟠 Высокие риски

R3. «Согласование» — 18 дней внешней зависимости

Самый длинный этап требует участия стейкхолдеров. Типичное бутылочное горлышко аналитической фазы: отпуск согласующего в августе → +2–3 недели к расписанию.

📋 Митигация:

  • Декомпозировать «Согласование» на 3 этапа по 5–6 дней с промежуточными ревью
  • До 27.07 согласовать календарь доступности всех стейкхолдеров на август
  • Назначить ответственного за эскалацию при задержке >2 дней

R4. Единственная точка отказа

Всего 2 задачи + 1 веха. Один исполнитель = весь проект. Болезнь, отпуск, перегрузка → полная остановка.

📋 Митигация:

  • Назначить backup-исполнителя с долей загрузки ≥30% на период проекта
  • Документировать требования в OpenProject (bus factor >1)
  • При расширении WBS распределить задачи между минимум 2 участниками

🟡 Средние риски

R5. Отсутствует базовый план (Baseline)

Нет зафиксированного базового плана → нельзя применять Earned Value Management: SV = EV − PV, SPI = EV / PV не рассчитать. ΔFinish относительно Baseline неизвестен.

📋 Митигация:

  • Зафиксировать Baseline в OpenProject до 27.07 (команда: Создать baseline)
  • Настроить еженедельный мониторинг: SV, SPI, ΔFinish
  • При отклонении SPI <0.9 — запускать анализ причин и корректирующие действия

R6. Неполный охват фаз

Определена только фаза «Аналитика». Что после 01.09? Разработка? Тестирование? Проект выглядит незавершённым — либо это подпроект, либо фазы не внесены в WBS.

📋 Митигация:

  • Провести сессию по уточнению полного жизненного цикла проекта
  • Добавить в WBS фазы: Разработка → Тестирование → Внедрение
  • Определить контрольные вехи за 01.09 с привязкой к общему roadmap
