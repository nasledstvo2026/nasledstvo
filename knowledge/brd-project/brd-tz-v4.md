## 4 OpenClaw-агента

| Агент | Роль | Ключевой навык |
|-------|------|----------------|
| 🤖 **Controller** (оркестратор) | Точка входа, управляет пайплайном | Принимает триггер «Создать БТ», запускает Questioner → передаёт лог Compiler-у → передаёт драфт Verifier-у → выдаёт результат |
| 🧠 **Questioner** (интервьюер) | Старший бизнес-аналитик | 5 Whys + Не верь пользователю + 3 сущности (Симптом/Барьер/Потери) + SPIN + JTBD |
| 📝 **Compiler** (аналитик) | Системный архитектор | Математическая формула RCA (Симптом→Барьер→Корень) + Web Search (SearXNG) + Impact Mapping |
| 🛡️ **Verifier** (аудитор) | Логический аудитор / Red Teaming | RCA Integrity (причина→цель→метрика как единое уравнение) + SMART + INVEST + Лингвистический комплаенс формулы |

## Архитектура передачи

```
[Пользователь] → «Создать БТ» → Controller
    ↓
Controller → sessions_send → Questioner (с биндингом к чату)
    ↓ Questioner общается с пользователем (5-20 вопросов)
    ↓ Лог интервью (.md)
Controller → sessions_send → Compiler
    ↓ Compiler ищет контекст (Web Search) + формулирует RCA
    ↓ Драфт BRD (.md)
Controller → sessions_send → Verifier
    ↓ Verifier проверяет RCA Integrity + SMART + формулу
    ↓ Если баги → return Compiler-у (макс 2 итерации)
    ↓ ✅ Аппрув (.md)
Controller → конвертация в .docx → выдача пользователю в чат
```

## Ключевые системные промпты

### Questioner — правило «Не верь пользователю»
- Пользователи в 99% случаев НЕ знают свою проблему
- Игнорировать требования сделать фичу — вытаскивать контекст процесса
- Собрать 3 сущности: СИМПТОМ → БАРЬЕР → ПОТЕРИ (в цифрах)
- 5 Whys мягко: «Что происходит сразу после того, как...»
- Анонс длительности на старте, микро-резюме, право на «стоп»

### Compiler — математическая формула RCA
Формула раздела «Описание проблемы»:
> «[Симптом и измеримые Потери] происходит из-за [Барьер], что вызвано корневой причиной: [Корень].»
- Few-Shot: 2 примера (чат-бот, интерфейс бухгалтера)

### Verifier — логический аудитор
1. **RCA Integrity:** Проблема → Цель → Метрика как единое уравнение
2. **Охота на симптомы:** бракует, если проблема = человеческий фактор, а не системная причина
3. **Лингвистический комплаенс:** проверяет наличие связок «происходит из-за», «что вызвано корневой причиной»

## Типы агентов (по ответам Кирилла)
- **Controller:** отдельный агент-оркестратор (не Лунт)
- **Questioner:** отдельный агент с биндингом к этому же чату
- **Compiler:** отдельный агент, через sessions_send
- **Verifier:** отдельный агент, через sessions_send
