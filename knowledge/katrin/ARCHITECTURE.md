## 🧩 Состав продукта

### Агенты (2 шт.)

#### 1. katrin-agent — фронтальный агент
- **Роль:** отвечает Катрин (+7 311 760 717) в Telegram. Консультирует по 44-ФЗ/223-ФЗ, генерирует .docx, общается с человеком
- **Binding:** `match.channel=telegram, peer.id=311760717` — все сообщения от Катрин идут сюда, минуя main-агента
- **Модель:** `deepseek/deepseek-chat`
- **Workspace:** `~/.openclaw/agents/katrin-agent/workspace`
- **AGENTS.md:** прописаны обязанности, формат выдачи, границы, обязательная верификация через auditor-agent

#### 2. auditor-agent — верификатор (внутренний)
- **Роль:** асинхронная проверка юридических фактов и черновиков документов. Только критика, без генерации
- **Доступ:** 🔒 **Только от katrin-agent через sessions_send**. Любой запрос от другого агента или напрямую от пользователя игнорируется с отказом
- **Модель:** `deepseek/deepseek-chat`
- **Workspace:** `~/.openclaw/agents/auditor-agent/workspace`
- **AGENTS.md:** строгий формат вердикта, проверка отправителя, тематика 44-ФЗ/223-ФЗ, ЛП/МИ

### Связь между агентами
```
Катрин (Telegram) → katrin-agent → auditor-agent (sessions_send) → вердикт → Катрин
```
- `agentToAgent.enabled = true` в конфиге
- katrin-agent **обязан** перед выдачей ответа отправить юридические факты в auditor-agent
- Если auditor-agent недоступен — ответ с пометкой «без аудита»

## 🔒 Изолированные сессии (что это и зачем)

**Изолированная сессия (isolated)** — однократный, эфемерный запуск агента:
- **Нет истории чата** — каждый запуск с чистого листа
- **Не привязан к пользователю** — не умеет отвечать в Telegram, только выполнить задачу и завершиться
- **Работает по промпту** — получает чёткое ТЗ на один раз
- **Результат** — файл/отчёт/JSON, публикуется через announce или файл

**Для чего используется:**
- Регулярные задачи по расписанию (cron) — парсинг, генерация отчётов, проверки
- Фоновые операции без диалога с человеком
- Экономия контекста — без истории предыдущих запусков

**В ФармТендеролог:**
- Cron-задачи (07:00, 07:20, 07:30) — все isolated ✅
- Сам katrin-agent — **постоянная сессия** (история диалога с Катрин)
- auditor-agent — **постоянная сессия** (вызовы через sessions_send от katrin-agent)

## 💡 Принципы проектирования

1. katrin-agent — единственный, кто общается с человеком
2. auditor-agent — внутренний сервис, без доступа пользователям
3. Любой юридический факт проходит верификацию перед выдачей
4. Все cron-задачи завершаются до начала рабочего дня Катрин (08:00)
5. Отказ от внешних сервисов (LegalMCP) в пользу прямых источников (consultant.ru, ЕИС)
6. Публичные страницы не содержат имён пользователей
