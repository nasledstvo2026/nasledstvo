# Продукт: 44-ФЗ / 224-ФЗ — закупки ЛП и МИ

## Описание
Цифровой продукт для Катрин (@Su_Ri_Mi) — консультации по 44-ФЗ, 224-ФЗ, закупкам лекарственных препаратов и медицинских изделий. Позволяет агенту отвечать с точными ссылками на закон, используя MCP-сервер российского законодательства.

```
[Катрин в Telegram]
       ↓ вопрос по 44-ФЗ/224-ФЗ/закупкам
[Лунт (OpenClaw)]
       ↓ перед ответом читает
┌─────────────────────────────────────┐
│  knowledge/katrin/zakupki-knowledge.md  │ ← база знаний (общие положения)
│  knowledge/katrin/weekly-update.md      │ ← актуальные изменения (последний cron)

└─────────────────────────────────────┘
       ↓ если нужна точная статья закона
[LegalMCP (MCP-сервер)]
       ↓ https://legalmcp.ru/mcp
[База: 13 кодексов, 9000+ ФЗ, постановления]
```

## Компоненты

- Основные положения 44-ФЗ, 224-ФЗ, ПП №620, ПП №1875

### 2. Промпт сессии (`knowledge/katrin/prompt-katrin.md`)

- Обязанность читать базу знаний перед ответом

### 3. Cron-обновление (пн/ср/пт 09:30)
- Скрипт: `scripts/legalmcp-update.py`
- Подключается к LegalMCP, делает 7 запросов
- Сохраняет конспект в `knowledge/katrin/weekly-update.md`
- Критические изменения → алерт Кириллу в Telegram

### 4. LegalMCP (MCP-сервер)
- URL: `https://legalmcp.ru/mcp`
- Токен: `lmcp_n…HPF4` (в TOOLS.md)
- Лимит: 100 запросов/мес бесплатно
- Расход: 84/мес (7 запросов × 3 раза/нед × 4 нед)

- Зафиксирован в Skill Workshop (applied 28.06.2026)

## Цепочка восстановления при ошибках

### ❌ «LegalMCP не отвечает / ошибка соединения»
1. Проверить статус: `python3 scripts/legalmcp-update.py` (тестовый запуск)
2. Если краш — проверить код/зависимости в скрипте
3. Если сервер недоступен — проверить https://legalmcp.ru
4. Fallback: ответить из существующей базы знаний, указав что изменения не проверены

### ❌ «LegalMCP краулер устарел (stale коллекции)»
1. Отправить `report_missing_data` — feedback loop через MCP
2. Или написать в поддержку LegalMCP
3. Пока ждём — пользоваться pravo.gov.ru вручную через web_fetch
4. Либо перейти на платный тариф (если проблема с приоритетом)

### ❌ «Превышен лимит LegalMCP (100/мес)»
1. Проверить кол-во запросов: токены на https://legalmcp.ru/dashboard
2. Если близко к лимиту — временно сократить частоту cron до 1 раза/нед
3. Если превышен — либо ждать нового месяца, либо подключать платный тариф (9 900 ₽/мес)
4. Альтернатива: интеграция с платным API КонсультантПлюс / Гарант

### ❌ «Cron-задача упала / не выполнилась»
1. Проверить логи: `openclaw tasks` / логи Gateway
2. Если скрипт упал — запустить вручную: `cd ~/.openclaw/workspace && python3 scripts/legalmcp-update.py`
3. Если проблема в токене — обновить в TOOLS.md
4. Если проблема в конфиге cron — пересоздать задачу

### ❌ «Ответ Катрин содержит неактуальные данные»
1. Проверить дату последнего обновления в `knowledge/katrin/weekly-update.md`
2. Запустить обновление вручную (скрипт выше)
3. Если изменение свежее (вчера/сегодня) — проверить через recherche в LegalMCP вручную
4. При подтверждении — дописать в weekly-update.md вручную
