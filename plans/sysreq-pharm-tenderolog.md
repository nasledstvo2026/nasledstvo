## 1. Архитектура системы

### 1.1. Компоненты

| Компонент | Описание | Технология |
|-----------|----------|-----------|
| **katrin-agent** | Основной агент: генерация и исправление документов | OpenClaw persistent-сессия (agent id: katrin-agent) |
| **auditor-agent** | Агент-аудитор: критика документов на compliance | OpenClaw persistent-сессия (agent id: auditor-agent) |
| **LegalMCP** | MCP-сервер для юридической нормативной базы | MCP API (lmcp_npXn...) |
| **Telegram-канал** | Интерфейс пользователя | OpenClaw Gateway → custom channel |
| **python-docx** | Генерация .docx | Библиотека Python |
| **Источники данных** | УФАС, ЕИС, ЖНВЛП | API / парсер (to be researched) |

### 1.2. Сеть и развёртывание

```
[Telegram: Катрин] → [OpenClaw Gateway]
                            ↓
            ┌──────────────────────────────┐
            │     session:katrin            │
            │     (katrin-agent)            │
            │     Генерация + Исправление   │
            └──────────┬───────────────────┘
                       │ sessions_send
            ┌──────────▼───────────────────┐
            │     session:auditor           │
            │     (auditor-agent)           │
            │     Критика / Аудит           │
            └──────────────────────────────┘
```

## 2. Агенты

### 2.1. katrin-agent
- **ID:** `katrin-agent`
- **Сессия:** `session:katrin` (persistent)
- **Модель:** deepseek-chat
- **Скилл:** `katrin-tender-expert`
- **AGENTS.md:** Тендерный эксперт, генератор документов
- **Память:** `memory/katrin/`
- **Знания:** `knowledge/katrin/` (НПА, процедуры, образцы)
- **Инструменты:** read, edit, write, web_search, web_fetch, exec, sessions_send
- **Cron:** пн/ср/пт 09:30 — обновление через LegalMCP

**Ответственность:**
- Приём запросов от Катрин через Telegram
- Генерация черновиков документов (.docx, python-docx)
- Отправка на аудит: `sessions_send(sessionKey="session:auditor", message=...)`
- Исправление документа на основе критики
- Отдача готового файла Катрин

### 2.2. auditor-agent
- **ID:** `auditor-agent`
- **Сессия:** `session:auditor` (persistent)
- **Модель:** deepseek-chat
- **Скилл:** `compliance-audit` (предстоит создать)
- **AGENTS.md:** Юрисконсульт-аудитор, compliance-специалист
- **Память:** `memory/auditor/`
- **Знания:** `knowledge/auditor/` (УФАС-практика, реестры, комплаенс-правила)
- **Инструменты:** read, edit, web_search, web_fetch, exec

**Ответственность:**
- Только критика и аудит (не генерирует документы)
- Проверка соответствия: 44-ФЗ, 224-ФЗ, ПП №620, ПП №1875
- Проверка по практике УФАС
- Сверка с реестром контрактов ЕИС
- Сверка с реестром предельных цен ЖНВЛП
- Формирование отчёта: [Нарушение / Риск / OK]

## 3. Протокол взаимодействия агентов

### 3.1. Межсессионная связь
```python
# katrin-agent → auditor-agent
response = sessions_send(
    sessionKey="session:auditor",
    message=f"""
    Проведи аудит документа.
    Тип: {doc_type}
    Контекст: {context}
    Текст документа: {doc_text}
    Специфические риски: {risks}
    """,
    timeoutSeconds=120
)
```

### 3.2. Цикл аудита
1. **Генерация:** katrin-agent создаёт черновик (python-docx)
2. **Отправка:** katrin-agent шлёт auditor-agent (текст + метаданные)
3. **Аудит:** auditor-agent проверяет по всем источникам
4. **Ответ:** auditor-agent возвращает список [замечание / риск / норма]
5. **Исправление:** katrin-agent вносит правки
6. **Финал:** katrin-agent отдаёт .docx Катрин + комментарий по правкам

### 3.3. Fallback
- Если auditor-agent не отвечает (timeout 120 сек) → katrin-agent выдаёт документ с пометкой «без аудита»
- Если auditor-agent вернул ошибку → katrin-agent извещает Катрин, предлагает повторить

## 5. Формат документов

- **Файлы вывода:** `.docx` (Microsoft Office Open XML)
- **Стандарт:** python-docx 1.1+
- **Шаблоны:** Предсозданные .docx-шаблоны в `templates/katrin/`
- **Запрещено:** `.md`, `.txt`, `.html` как финальный формат для Катрин

## 6. Безопасность

- Оба агента — persistent-сессии OpenClaw, изолированы друг от друга
- Межсессионная связь — только через OpenClaw Gateway (sessions_send)
- Никаких внешних API-вызовов между агентами
- Доступ к данным — только из памяти/знаний соответствующего агента
- Катрин видит только katrin-agent (Telegram-маршрутизация)

## 7. Мониторинг

| Метрика | Источник | Периодичность |
|---------|----------|--------------|
| Количество сгенерированных документов | katrin-agent memory | Ежедневно |
| Количество замечаний аудита на документ | auditor-agent memory | По документу |
| Успешность циклов аудита (успех/таймаут/ошибка) | Logs Gateway | Ежедневно |
| Расход LegalMCP (вызовы/месяц) | legalmcp-update.py stats | пн/ср/пт |
