## 1. Целевая архитектура

```
[Telegram: Катрин] → [OpenClaw Gateway через кастомный channel или второго бота]
                             ↓
                    session:katrin (persistent-агент)
                        ├─ Скилл: katrin-tender-expert
                        ├─ Модель: deepseek-chat (или v4-flash — решить при старте)
                        ├─ Своя память: memory/katrin/
                        ├─ Свои знания: knowledge/katrin/ (уже готовы)
                        ├─ Инструменты: read, edit, write, web_search, web_fetch, exec
                        └─ Cron: пн/ср/пт 09:30 — обновление LegalMCP
```

### Шаг 5. План отката

| Ситуация | Действие |
|----------|----------|
| Катрин не получает ответы | Переключить маршрутизацию обратно на main. Восстановить старый cron |
| LegalMCP-крон падает | Откат крона на main, разбираться отдельно |
| Контекст путается, агент отвечает не по теме | Очистить память сессии, проверить AGENTS.md, перезапустить |
| Полный провал | Отключить session:katrin, удалить кастомный маршрут, вернуть всё как было. Длительность отката: ~5 минут |
