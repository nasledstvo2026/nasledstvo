О проекте

«Создать БТ» — сервис, который решает проблему низкой культуры работы с бизнес-требованиями. Владельцы продуктов и бизнес-аналитики тратят много времени, используют разные шаблоны, формулируют поверхностные требования без анализа корневых причин.

Сервис заменяет статический шаблон и пустой документ на динамический диалог с AI-агентом в Telegram. Пользователь не заполняет форму — он разговаривает с экспертом, который слушает, уточняет, углубляется, контролирует качество — и в итоге выдаёт готовый документ.

Мультиагентная архитектура

В основе — 4 специализированных AI-агента, работающих последовательно через единый Session Context:

  • Агент-опросчик — ведёт диалог, задаёт вопросы по 7 блокам шаблона, адаптирует глубину (L1→L2→L3) в зависимости от полноты ответов
  • Агент-составитель — формирует текст БТ по утверждённому шаблону без лишней художественности
  • Агент-контролёр — проверяет каждый раздел на полноту и глубину; при поверхностном ответе возвращает на доработку
  • Агент-генератор — создаёт DOCX и веб-версию, публикует на GitHub Pages

Канал взаимодействия: Telegram (основной) → мультиагентный пайплайн на OpenClaw + DeepSeek API → генерация DOCX (python-docx) → публикация на GitHub Pages.

Безопасность: аутентификация по Telegram user ID, белый список пользователей (whitelist.json), шифрование сессий AES-256, Cloudflare Tunnel (HTTPS), API-ключи в переменных окружения.

Ключевые отличия

  • Динамическая глубина: при поверхностном ответе агент задаёт уточняющие вопросы, а не принимает «как есть». Три уровня: L1 (открывающий) → L2 (углубляющий) → L3 (корневой)
  • Контроль качества в реальном времени: контролёр проверяет каждый раздел на полноту и возвращает на доработку, если блок заполнен поверхностно
  • Самообучаемость: с каждым новым БТ сервис накапливает опыт — анализирует паттерны вопросов и ответов, качество опроса растёт от сессии к сессии
  • Единый формат: утверждённый шаблон из 7 блоков, неменяемый после ввода в эксплуатацию
  • Двойной формат выдачи: DOCX (для Jira) + веб-страница на GitHub Pages — идентичное содержание
  • Жёсткий шаблон: без лишней художественности, только по делу

Шаблон БТ (7 блоков)

Агент последовательно проходит по каждому блоку, задавая вопросы с динамической глубиной. При полном ответе — фиксирует и идёт дальше. При поверхностном — уточняет:

  • Блок 1 — Контекст и проблема: контекст инициативы → проблемная область → последствия бездействия
  • Блок 2 — Цель и измерение: бизнес-цель → критерии успеха (SMART)
  • Блок 3 — Стейкхолдеры и влияние: заинтересованные стороны → их интересы и влияние
  • Блок 4 — Решение: ключевые функциональные возможности → пользовательские сценарии → граничные случаи
  • Блок 5 — Требования к качеству и ограничения: качественные характеристики → ограничения
  • Блок 6 — Приёмка: критерии приёмки (Given-When-Then)
  • Блок 7 — Риски и митигации: идентифицированные риски (собираются в течение всего диалога) → митигации

Пользователи

Первичные: владельцы продуктов (Катя, Лена, Данил, Роман, Кирилл) и бизнес-аналитики — инициируют новые инициативы, формируют БТ.

Вторичные: разработчики (получают чёткие требования) и руководители (оценивают трудозатраты по документированным данным).

Эпики (5 эпиков, 28 историй)

Эпик 1 — Управление доступом и сессиями

7 историй · FR1–FR5, FR26–FR28. Начало и продолжение сессии, автосохранение черновиков, белый список, изоляция данных, шифрование.

Эпик 2 — Динамический опрос по шаблону БТ

7 историй · FR6–FR11. Адаптивный опрос по 7 блокам, L1→L2→L3, сбор рисков, возврат к блокам.

Эпик 3 — Контроль качества и черновик

6 историй · FR12–FR16. Составитель, контролёр, проверка полноты, возврат на доработку.

Эпик 4 — Генерация и публикация

5 историй · FR17–FR21. DOCX (python-docx), веб-версия, GitHub Pages, ссылки в Telegram.

Эпик 5 — Самообучение

4 истории · FR22–FR25. history.json, анализ паттернов, повышение качества от сессии к сессии.

Инфраструктура

  • VPS vm-f13581 (Linux) — хостинг сервиса
  • OpenClaw — оркестрация мультиагентного пайплайна
  • DeepSeek API — LLM для работы всех агентов
  • Telegram Bot API — канал взаимодействия с пользователем
  • Cloudflare Tunnel — HTTPS-доступ к VPS
  • GitHub Pages — публикация DOCX и веб-версий
  • python-docx — генерация DOCX-файлов

MVP Scope

Входит: Telegram-бот, динамический опрос по 7 блокам, мультиагентный пайплайн (опросчик → составитель → контролёр → генератор), генерация DOCX, публикация на GitHub Pages, хранение истории сессий, самообучаемость.

Вне MVP: интеграция с Jira, мультиязычность, веб-интерфейс управления сессиями, аналитическая панель, Party Mode, Advanced Elicitation.

Целевые метрики

  • Полнота разделов БТ: 7/7 блоков заполнены
  • Время ответа агента: <5 сек (95-й перцентиль)
  • Генерация DOCX: <30 сек
  • Публикация на GitHub Pages: <60 сек
  • Доступность: 99% в рабочее время
  • Одновременных сессий: до 20
  • Хранение сессий: не менее 500 завершённых БТ
