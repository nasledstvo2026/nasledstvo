## 1. УФАС — База решений (br.fas.gov.ru)

| Параметр | Значение |
|----------|----------|
| **URL** | https://br.fas.gov.ru/ |
| **Открытое API?** | Нет документированного REST API |
| **Веб-доступ** | ✅ Да — HTML-страница с фильтрами, пагинацией, поиском |
| **Парсинг** | Возможен: HTML → web_fetch по URL с параметрами (?page=N, ?type=, поиск по номеру дела) |
| **Кейс для auditor-agent** | Проверка решения УФАС по аналог. кейсам (жалобы на 44-ФЗ/223-ФЗ) |
| **Подход** | `web_fetch` + `web_search` для поиска релевантных решений |

**Вердикт:** Доступен через web_fetch + web_search. Парсер не требуется — агент может искать сам.

## 2. ЕИС — zakupki.gov.ru

| Параметр | Значение |
|----------|----------|
| **URL** | https://zakupki.gov.ru/ |
| **Открытое API?** | ✅ SOAP-сервисы отдачи данных через int44.zakupki.gov.ru |
| **FTP** | ❌ Закрыт с 01.01.2025 |
| **Требуется** | Токен (через Госуслуги) или ЭЦП для подписи SOAP-запросов |
| **Формат** | XML через SOAP-протокол |
| **Парсинг** | Сложный: SOAP-клиент, XML-схемы, токен/ЭЦП |
| **Кейс для auditor-agent** | Проверка контракта в реестре по ИКЗ / номеру |
| **Подход** | `web_search` + `web_fetch` по открытой странице zakupki.gov.ru (реестр контрактов доступен через HTML). SOAP — сложно, неоправданно для MVP |

**Вердикт:** Полноценное API — только через SOAP с токеном/ЭЦП. Для MVP — web_fetch открытых HTML-страниц реестра контрактов.

| Источник | Доступность | Тип доступа | Сложность MVP | Использовать в Фазе 3? |
