## 3. Поток данных: Генерация документа с аудитом

```
Специалист (Telegram):
  ┌─ "Создай допсоглашение к контракту №ХХХ на поставку ЛП..."
  │
  ▼
┌──────────────────────────────────────────────────────────────────┐
│                  katrin-agent (session:katrin)                    │
│                                                                   │
│  1. Принимает запрос                                              │
│  2. Парсит Intent (тип документа, контракт, препараты, цена)      │
│  3. Читает знания knowledge/katrin/ — типовые формы, НПА          │
│  4. LegalMCP — актуальные нормы по запросу                        │
│  5. Генерирует черновик (python-docx → .docx)                     │
│  6. Сериализует текст + метаданные для аудита                     │
│                                                                    │
│  ┌─ sessions_send("session:auditor", "...") ───────────────────┐  │
│  │  {                                                          │  │
│  │    "doc_type": "dop_soglashenie",                           │  │
│  │    "context": "контракт №ХХХ, ЛП: Аспирин-Кардио, 500уп",  │  │
│  │    "doc_text": "...",                                       │  │
│  │    "risks": ["цена ЖНВЛП", "изменение существенных условий"] │  │
│  │  }                                                          │  │
│  └─────────────────────────────────────────────────────────────┘  │
│                                                                    │
│  ◀── Ответ auditor-agent: список замечаний                        │
│                                                                    │
│  7. Вносит исправления в черновик на основании замечаний auditor-agent (python-docx) │
│  8. Формирует финальный .docx                                     │
│  9. Отправляет специалисту: файл + краткий комментарий по правкам      │
└──────────────────────────────────────────────────────────────────────┘
                    │ sessions_send
                    ▼
┌──────────────────────────────────────────────────────────────────┐
│                  auditor-agent (session:auditor)                  │
│                                                                   │
│  1. Принимает запрос на аудит                                     │
│  2. Проверяет 44-ФЗ/224-ФЗ через LegalMCP или knowledge/auditor/ │
│  3. Проверяет практику УФАС (по источнику — когда будет готов)    │
│  4. Сверяет типовые формы с реестром контрактов ЕИС               │
│  5. Сверяет цены с реестром предельных цен ЖНВЛП                  │
│                                                                    │
│  6. Формирует отчёт:                                              │
│  ┌──────────────────────────────────────────────────────────────┐ │
│  │ [НАРУШЕНИЕ] П.2 ст.95 44-ФЗ: изменение сущ. условий без     │ │
│  │             основания                                        │ │
│  │ [РИСК]     Цена Аспирин-Кардио превышает реестр ЖНВЛП       │ │
│  │ [OK]       Форма допсоглашения соответствует ПП №620         │ │
│  └──────────────────────────────────────────────────────────────┘ │
│                                                                    │
│  7. Возвращает katrin-agent: отчёт + рекомендации                  │
└──────────────────────────────────────────────────────────────────────┘
```
