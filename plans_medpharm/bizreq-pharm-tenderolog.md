# Бизнес-требования: Цифровой продукт «ФармТендеролог»

> Версия: 1.0
> Дата: 29.06.2026
> Статус: Черновик

---

## 1. Концепция продукта

«ФармТендеролог» — интеллектуальный юридический ассистент для участников закупок лекарственных препаратов (ЛП) и медицинских изделий (МИ) по 44-ФЗ и 224-ФЗ.

Продукт автоматизирует создание юридически корректных документов (допсоглашения, закупочная документация) с многоуровневой проверкой на соответствие нормативной базе, практике УФАС и реестрам предельных цен.

## 2. Проблема

| Проблема | Кого касается |
|----------|--------------|
| Подготовка допсоглашения требует сверки с 44-ФЗ, ПП №620, ПП №1875, Честным ЗНАКом — вручную это часы работы юриста | Катрин, тендерные специалисты |
| Юридическая ошибка в документе = риск отклонения контролирующими органами, жалоба в УФАС, срыв контракта | Катрин, поставщики ЛП/МИ |
| Необходимо отслеживать актуальную практику УФАС и предельные цены ЖНВЛП — без автоматизации это постоянный ручной мониторинг | Юридический отдел |
| Разрозненные источники: pravo.gov.ru, zakupki.gov.ru, fas.gov.ru — нет единого окна проверки | Команда закупок |

## 3. Целевая аудитория

| Роль | Потребность |
|------|------------|
| **Катрин** (тендерный специалист) | Быстро генерировать корректные документы, не тратить часы на сверку с НПА |
| **Руководитель (Кирилл)** | Контролировать качество документов и эффективность процессов |
| **Юридический отдел** (будущий) | Аудит документов на compliance, доступ к практике УФАС |

## 4. Пользовательские сценарии (User Stories)

1. **Как тендерный специалист**, я хочу отправить запрос на допсоглашение — и получить готовый .docx, чтобы не тратить время на написание вручную.
2. **Как тендерный специалист**, я хочу чтобы документ автоматически проверялся на соответствие 44-ФЗ, чтобы исключить ошибки.
3. **Как тендерный специалист**, я хочу чтобы проверка включала практику УФАС по аналогичным кейсам, чтобы не получить жалобу.
4. **Как тендерный специалист**, я хочу чтобы при изменении цены препарата система сверялась с реестром предельных цен ЖНВЛП.
5. **Как руководитель**, я хочу видеть метрики: сколько документов создано, сколько ошибок найдено аудитом.
6. **Как юридический отдел**, я хочу получать отчёт аудита по каждому документу с указанием конкретных норм и рисков.

## 5. Бизнес-ценность

| Метрика | Ожидаемый эффект |
|---------|-----------------|
| Время на подготовку допсоглашения | Сокращение с ~2-3 часов до 15-20 минут |
| Доля документов с юр. ошибками | Снижение до <5% (с ручных ~30-50%) |
| Риск жалобы в УФАС | Минимизация за счёт сверки с практикой |
| Скорость реакции на изменения в НПА | Автоматическое обновление знаний (LegalMCP) |

## 6. Стейкхолдеры

| Стейкхолдер | Роль в продукте |
|-------------|----------------|
| **Кирилл** | Владелец продукта, принимает решения по развитию |
| **Катрин** | Основной пользователь, формулирует потребности |
| **Лунт** | Архитектор и разработчик продукта |
| **Юридический отдел** (перспектива) | Внутренний аудитор качества документов |

## 7. Ограничения

- Бесплатный лимит LegalMCP — 100 запросов/месяц
- Открытые API УФАС, ЕИС, ЖНВЛП — требуется исследование на доступность
- Формат документов — только .docx (Microsoft Office), никакого Markdown, никаких .md-шаблонов
- Infra: GitHub Pages + VPS + Cloudflare Tunnel (без кастомного домена)

## 8. Критерии успеха (MVP)

- [ ] katrin-agent отвечает на вопросы по 44-ФЗ/224-ФЗ с цитатами НПА
- [ ] katrin-agent генерирует .docx по запросу
- [ ] auditor-agent проверяет документ на соответствие нормам
- [ ] Цикл «генерация → аудит → исправление → отдача» работает в один проход
- [ ] Telegram-маршрутизация: сообщения Катрин приходят только katrin-agent
