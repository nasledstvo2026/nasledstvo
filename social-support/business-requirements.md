## Блок 1. Контекст и проблема

### 1.1 Контекст инициативы
**Проект:** «Мера социальной поддержки»  
**Суть:** Поддержка в актуальном состоянии исчерпывающего перечня мер социальной поддержки для трёх категорий граждан РФ:
- Люди с инвалидностью
- Ветераны боевых действий
- Участники СВО и члены их семей

Источник — нормативно-правовые акты и официальные государственные источники. Охват: федеральные и региональные меры поддержки.

### 1.2 Проблемная область
Актуальность данных поддерживается **вручную**. Это трудозатратно, медленно и чревато ошибками. Изменения в законодательстве происходят регулярно (индексации выплат, новые постановления, утрата силы старых), и ручной мониторинг не поспевает.

### 1.3 Последствия бездействия
- **Ошибки в данных:** неверные суммы, пропущенные/устаревшие меры поддержки
- **Репутационные потери:** граждане получают недостоверную информацию
- **Жалобы и обращения:** от пользователей, которым система выдала неверные данные
- **Потеря времени:** ручная перепроверка каждого НПА отнимает часы еженедельно
- **Юридические риски:** неактуальные ссылки на отменённые НПА
- **Упущенные меры:** новые выплаты/льготы не попадают в перечень вовремя

### 4.2 Особенности предметной области по категориям

#### 4.2.1 Ветераны боевых действий (ВБД)
**Базовый документ:** Федеральный закон от 12.01.1995 № 5-ФЗ «О ветеранах», статья 3.

Меры поддержки **зависят от подкатегории** ветерана. В соответствии со ст. 16 5-ФЗ выделяют три группы:

1. **ВБД из числа лиц, указанных в подпунктах 1–4, 8 п. 1 ст. 3 5-ФЗ** — участники боевых действий:
   - сотрудники госбезопасности, ОВД, Минобороны, ФСБ, Нацгвардии, Следственного комитета, прокуратуры, противопожарной службы, уголовно-исполнительной системы
   - участники разминирования территорий (1945–1951)
   - участники отражения нападения на Дагестан (1999) в составе отрядов самообороны
   - добровольцы, участвующие в СВО
   - участники военных операций в Афганистане
   - участники военных операций на территории ДНР и ЛНР с 11.05.2014
   - лица, заключившие контракт с частными военными компаниями, содействующими ВС РФ в ходе СВО

2. **ВБД из числа лиц, указанных в подпункте 5 п. 1 ст. 3 5-ФЗ** — гражданский персонал воинских частей ВС СССР и ВС РФ на территориях других государств в период боевых действий, получившие ранение либо удостоенные наград

3. **ВБД из числа лиц, указанных в подпунктах 6, 7 и 9 п. 1 ст. 3 5-ФЗ** — лица, не состоящие на военной службе, направленные для выполнения профессиональных задач в зону боевых действий, отработавшие установленный срок либо откомандированные досрочно по уважительной причине

**Система должна** при анализе НПА определять, к какой подкатегории ВБД относится мера, и явно указывать это в карточке меры.

#### 4.2.2 Люди с инвалидностью
**Базовый документ:** Федеральный закон от 24.11.1995 № 181-ФЗ «О социальной защите инвалидов».

Меры различаются по **группам инвалидности** (I, II, III). Имеются **специфические меры** для:
- детей-инвалидов
- инвалидов по отдельным причинам инвалидности (военная травма, чернобыльцы, профессиональные заболевания и т.д.)

**Система должна** учитывать группу инвалидности и причину инвалидности при извлечении и отображении мер.

#### 4.2.3 Участники СВО и члены их семей
**Базовый пакет:** Указы Президента РФ, федеральные законы о статусе военнослужащих.

**Особенности:**
- меры **постоянно меняются** (высокая динамика изменений)
- часть мер адресована **членам семей**, а не самим участникам
- требуется различать получателя: участник СВО / член семьи (супруга, дети, родители)

**Система должна** отслеживать изменения с повышенной частотой и явно указывать получателя меры.

### 4.3 Пользовательские сценарии

**Happy path (типовой день Ирины):**

1. Ирина открывает **дашборд**
2. Видит три базы данных по категориям (инвалиды, ВБД, СВО)
3. Запускает **сверку** базы по выбранной категории с действующими НПА
4. Далее — автоматически по расписанию: поиск обновлений за прошедший период

**Что делает система:**
- Ищет изменения в НПА по приоритетным источникам (см. п. 4.5)
- Анализирует и извлекает меры с параметрами (название, тип, сумма, получатель, условия, сроки, основание)
- Для ВБД: определяет подкатегорию по ст. 3 5-ФЗ
- Для инвалидов: учитывает группу и причину инвалидности
- Для СВО: определяет получателя (участник / член семьи)
- Сверяет с текущей базой
- Помечает **новые** и **изменившиеся** меры

**Результат на дашборде Ирины:**
- Сводка: «Найдено N новых мер, M изменённых»
- По каждой мере — карточка с параметрами и ссылкой на пункт документа
- По изменённой — подсветка: что именно поменялось
- Если параметр не найден — честно: «Не указано в документе»

**Завершение:** Ирина проверяет, жмёт «Подтвердить» → база обновлена → задача закрыта.

### 4.4 Граничные случаи и исключения

| Сценарий | Действие системы |
|----------|------------------|
| **Поиск: документ не найден/недоступен** | Сообщает: «Недоступен. Проверьте ссылку или вручную». Даёт поле для ручной загрузки |
| **Анализ: документ не парсится** | Предупреждение с пометкой «Проверить вручную» |
| **Актуальность: источник недоступен** | 3 попытки → сообщение: «Источник временно недоступен, задача отложена» |
| **Актуальность: документ утратил силу** | Ищет замену. Если нет — помечает меры как недействующие, не удаляет, ставит «Документ утратил силу» + дата утраты силы |
| **Актуальность: мера отменена новым НПА** | Подсвечивает «Исключена» + дата исключения |
| **Актуальность: истёк срок действия меры** | «Срок истёк» + дата завершения |
| **Сверка: мера исчезла из нового НПА** | Подсвечивает. Не удаляет. Ирина решает: пометить недействующей или оставить с комментарием |
| **Сверка: параметр изменился** | Сравнение «было → стало» со ссылками на оба НПА, ждёт подтверждения |
| **Сверка: похоже на дубликат** | Показывает карточки рядом. Ирина выбирает: объединить, заменить или оставить обе |
| **Сверка: противоречие между документами** | Подсветка красным, показывает источники, **требует ручного разрешения** |
| **ВБД: подкатегория не определена** | Пометка «Подкатегория ВБД не установлена» + ссылка на ст. 3 5-ФЗ для ручной классификации |
| **Инвалидность: группа не указана** | Пометка «Группа инвалидности не указана в документе» |
| **СВО: получатель не указан явно** | Разбор контекста: если неясно — пометка «Требуется уточнение получателя» |
| **Региональная мера: источник не найден** | Пометка «Региональный источник недоступен», ссылка на официальный сайт региона |

**Главные правила:**
1. Автоматически ничего не удаляем — только по команде Ирины
2. Частичный результат отдаём, а не отбрасываем
3. О любой проблеме говорим прямо и предлагаем действие
4. Все изменения логируем: что, когда и на основании какого документа поменялось

### 4.5 Приоритет источников данных

Источники проверяются в порядке убывания приоритета:

| № | Источник | Тип | Область применения |
|---|----------|-----|--------------------|
| 1 | [garant.ru](https://www.garant.ru/) | Правовая система | Федеральные и региональные НПА |
| 2 | [docs.cntd.ru](https://docs.cntd.ru/) | Электронный фонд правовых и НТД | Федеральные и региональные НПА |
| 3 | [consultant.ru](https://www.consultant.ru/) | Правовая система | Федеральные и региональные НПА (резервный) |
| 4 | [sfr.gov.ru](https://sfr.gov.ru/) | Официальный сайт СФР | Выплаты, пенсии, ЕДВ, НСУ |
| 5 | Официальные сайты правительств регионов | Региональные порталы | Региональные меры поддержки |
| 6 | Официальные сайты региональных департаментов труда и соцзащиты | Ведомственные сайты | Региональные меры поддержки (детали) |
| 7 | [nalog.gov.ru](https://www.nalog.gov.ru/) | ФНС России | Налоговые льготы |

**Правила работы с источниками:**
- Приоритетный источник проверяется первым. Если данные найдены — остальные не опрашиваются
- Если приоритетный источник недоступен — переход к следующему по приоритету
- При противоречии между источниками — подсветка красным, ручное разрешение
- Результаты каждого запроса к источнику логируются

## Блок 7. Риски и митигации

### 7.1 Идентифицированные риски

| # | Риск | Категория | Вероятность | Влияние | Приоритет |
|---|------|-----------|-------------|---------|-----------|
| R1 | **Качество парсинга НПА** — разные форматы (PDF, скан, вёрстка), не всё парсится корректно | Технический | Высокая | Высокое | Критический |
| R2 | **Доступность источников** — официальные сайты могут быть недоступны, менять URL без предупреждения | Технический | Средняя | Среднее | Средний |
| R3 | **Частота изменений** — неравномерная: всплески индексаций, новые законы. Особенно критично для СВО | Бизнес | Средняя | Среднее | Средний |
| R4 | **Adoption** — Ирина привыкла к ручному процессу, может не доверять автоматизации | Организационный | Средняя | Среднее | Средний |
| R5 | **Сроки** — MVP за 5 дней, релиз за 16 дней, возможны задержки | Организационный | Высокая | Критическое | Критический |
| R6 | **Сложность категоризации** — ВБД имеют 3 подкатегории, инвалиды — группы и причины, СВО — разграничение получателей. Ошибка в классификации ведёт к неверным мерам | Предметный | Средняя | Высокое | Высокий |
| R7 | **Региональные меры** — 89 регионов с разными форматами публикации, нестандартной структурой данных | Технический | Высокая | Среднее | Высокий |

### 7.2 Митигации

| Риск | Митигация |
|------|-----------|
| **R1. Качество парсинга** | — Fallback: ручная загрузка документа при неудаче<br>— Пометка «Проверить вручную» для непрозрачных случаев<br>— Постепенное обучение на реальных документах |
| **R2. Доступность источников** | — 3 попытки с задержкой<br>— Переход к следующему по приоритету источнику<br>— Поле для ручной загрузки<br>— Логирование всех сбоев для анализа |
| **R3. Частота изменений** | — Гибкое расписание проверок с повышенной частотой для СВО<br>— Приоритизация категорий по частоте изменений |
| **R4. Adoption** | — Прозрачность: система показывает источники каждого решения<br>— Режим «только рекомендация» на старте<br>— Ирина всегда подтверждает изменения вручную |
| **R5. Сроки** | — MVP: минимальный набор функций (F1 + F2 по одной категории, без полноценного F3)<br>— Приоритет: работающий прототип > полная функциональность<br>— План отката: вернуться к ручному процессу при срыве сроков |
| **R6. Сложность категоризации** | — Явные справочники подкатегорий для каждой групп<br>— Валидация перед подтверждением<br>— Пометки при неопределённой категории |
| **R7. Региональные меры** | — Поэтапное подключение регионов (пилот — 1-2 региона)<br>— Универсальный шаблон карточки меры<br>— Fallback: ручной ввод для сложных регионов |
