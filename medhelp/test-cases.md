# MedHelp — Тест-кейсы приёмочного тестирования

> Продукт: Медицинский консультант для врачей (КТ, МРТ, ПЭТ/КТ)
> Версия требований: 0.1 (утверждена 07.07.2026)
> Дата: 07.07.2026
> Роль: Senior QA-инженер

---

## Легенда вердиктов

| Вердикт | Значение |
|---------|----------|
| ✅ | Факт подтверждён по обязательным источникам |
| ❌ | Факт противоречит обязательным источникам |
| 🔄 | Источники расходятся (Минздрав vs РОРР) — требуется решение врача |
| ❓ | Нет данных в российских источниках, проверено по международным |

### 1.2 Обработка ошибок и таймаутов

| ID | Название | Предусловия | Шаги | Ожидаемый результат | Критерий |
|----|----------|-------------|------|---------------------|----------|
| **TC-05** | Таймаут верификации (>45 сек) | medcheck-agent искусственно задержан (имитация высокой нагрузки). Минздрав и РОРР не отвечают в течение 45 сек | 1. Мария отправляет запрос по протоколу<br>2. Ожидание 45+ сек | 1. Через ~45 сек Мария получает промежуточное сообщение: «Проверка источников занимает больше времени, попробую позже. Ваш запрос сохранён.»<br>2. Ответ не содержит неверифицированных фактов<br>3. Сообщение не содержит технических деталей | 3, 5 |
| **TC-06** | Полная недоступность источников | Минздрав и РОРР недоступны (имитация сетевого отказа). Оба источника не отвечают | 1. Мария отправляет запрос по протоколу<br>2. Ожидание ответа | 1. Мария получает сообщение: «Источники верификации временно недоступны. Повторите запрос позже.»<br>2. Ответ НЕ содержит клинических рекомендаций (неверифицированная информация не выдаётся)<br>3. Общее время ответа ≤ 60 сек | 5, 8 |
| **TC-06b** | Частичная недоступность (один источник жив) | Минздрав недоступен, РОРР доступен | 1. Мария отправляет запрос по протоколу<br>2. Ожидание ответа | 1. Вердикт medcheck-agent: ❓<br>2. В ответе указано: «Проверено только по РОРР, Минздрав временно недоступен»<br>3. Ответ содержит цитаты из РОРР | 1, 5 |

## 3. Приёмочные тесты (должны быть пройдены для подтверждения MVP)

| ID | Название | Предусловия | Шаги | Критерий прохождения |
|----|----------|-------------|------|---------------------|
| **AT-01** | 20 запросов подряд — проверка полезности | Система запущена. Подготовлен список из 20 реалистичных запросов от Марии (разные протоколы, модальности, формулировки) | 1. Мария задаёт 20 вопросов последовательно<br>2. Каждый ответ проверяется:<br>   - Есть цитата/ссылка?<br>   - Есть вердикт?<br>   - Нет служебной информации?<br>   - Время ≤ 60 сек?<br>3. Мария даёт устную/письменную оценку | 1. Все 20 ответов содержат верифицированные данные<br>2. Доля ошибок ≤ 10% (ошибка = несоответствие Минздраву/РОРР)<br>3. Мария подтверждает: «Ассистент полезен в рутинной работе» | 6, 7 |
| **AT-02** | Статистика ошибок на 50 запросах | Система записала логи всех запросов | 1. Собрать 50 запросов (из AT-01 + предыдущие тесты)<br>2. Проверить каждый на соответствие источникам<br>3. Подсчитать долю ошибок | 1. Ошибок ≤ 5 из 50 (≤ 10%)<br>2. 🔄 не считается ошибкой<br>3. Результат задокументирован | 6 |

## 6. Пример детализированного теста (для автоматизации)

```
☐ Ответ содержит дословную цитату
```
