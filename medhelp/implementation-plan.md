## 0. Действия перед началом (критически важно)

### 0.1 Резервное копирование

Перед любыми изменениями конфигурации OpenClaw выполнить:

```bash
# 1. Бэкап текущего конфига
cp /home/user1/.openclaw/openclaw.json /home/user1/.openclaw/backups/openclaw.json.2026-07-07-pre-medhelp

# 2. Бэкап allowFrom
cp /home/user1/.openclaw/credentials/telegram-default-allowFrom.json \
   /home/user1/.openclaw/backups/telegram-default-allowFrom.json.2026-07-07-pre-medhelp

# 3. Бэкап USER.md
cp /home/user1/.openclaw/workspace/USER.md /home/user1/.openclaw/workspace/USER.md.2026-07-07-pre-medhelp

# 4. Бэкап всех агентов (если есть настройки katrin-agent)
cp -r /home/user1/.openclaw/agents /home/user1/.openclaw/backups/agents.2026-07-07-pre-medhelp

# 5. Бэкап workspace (проектная папка уже есть)
git add -A && git commit -m "backup: pre-medhelp state 2026-07-07" && git push
```

### 0.2 План отката (Rollback Plan)

**Катастрофический откат** — полный возврат к состоянию pre-MedHelp:

| Шаг | Команда |
|-----|---------|
| 1 | `openclaw gateway stop` |
| 2 | Восстановить openclaw.json из бэкапа |
| 3 | Восстановить allowFrom из бэкапа |
| 4 | `rm -rf /home/user1/.openclaw/agents/medhelp-agent /home/user1/.openclaw/agents/medcheck-agent` |
| 5 | Восстановить USER.md из бэкапа |
| 6 | `openclaw gateway start` |
| 7 | Проверить, что Мария идёт в main, а не в medhelp |
| 8 | Сообщить: «Откат выполнен. MedHelp не активирован.» |

**Некатастрофический откат** (проблема только в medcheck-agent):
- Убрать medcheck-agent из `agents.list` в конфиге → рестарт
- Мария продолжает получать консультации от medhelp-agent, но без верификации (ограниченный режим)

## Этап 1 — Архитектура и конфигурация (1 день)

### Цель: создать структуру агентов и настроить binding

### Действия

**1.1 Создать директории:**
```bash
mkdir -p /home/user1/.openclaw/agents/medhelp-agent/workspace
mkdir -p /home/user1/.openclaw/agents/medcheck-agent/workspace
mkdir -p /home/user1/.openclaw/workspace/logs/medhelp
```

**1.2 Написать AGENTS.md для medhelp-agent:**
- Роль: фронт-агент для Марии Комаровой
- Только Мария (peer binding)
- Помощник, не замена (не ставит диагнозы, не назначает лечение, не описывает снимки)
- Обязательная верификация через medcheck-agent (sessions_send)
- Контракт: JSON (claims + sources → verdict + details + verified_claims + sources_used)
- Ответ — связный текст, без служебной информации
- Таймауты: 45 сек на всю цепочку, 1 ретрай
- Память: MEMORY.yaml (preferences, frequent_queries, favorite_protocols)
- Skills: SKILLS.yaml, предлагать после 3 повторений
- Логи: 30 дней хранения

**1.3 Написать AGENTS.md для medcheck-agent:**
- Роль: изолированный верификатор
- Только medhelp-agent, всем остальным отказывать
- Не генерирует — только критика и верификация
- Источники: Минздрав > РОРР > Radiopaedia > ACR > НРБ-99/2009 > PubMed
- Правила: 2 обязательных источника, 🔄 при расхождении, ❓ если нет в РФ, ❌ при противоречии
- Кеш: Минздрав 2 нед, Radiopaedia 1 год, остальные 1 нед
- Таймаут: 30 сек на источник

**1.4 Написать SOUL.md:**
- medhelp-agent: профессиональный, уважительный. Помощник коллеги
- medcheck-agent: сухой, формальный. Только факты

**1.5 Написать TOOLS.md:**
- medhelp-agent: sessions_send → medcheck, web_search, web_fetch, r/w YAML
- medcheck-agent: web_search, web_fetch (приоритетные URL), кеш

**1.6 Обновить openclaw.json — добавить агентов в agents.list:**
```json
{
  "agents": {
    "list": {
      "medhelp-agent": {
        "agentDir": "/home/user1/.openclaw/agents/medhelp-agent",
        "model": "deepseek/deepseek-chat",
        "sessionTarget": "isolated",
        "agentToAgent": {"enabled": true}
      },
      "medcheck-agent": {
        "agentDir": "/home/user1/.openclaw/agents/medcheck-agent",
        "model": "deepseek/deepseek-chat",
        "sessionTarget": "isolated",
        "agentToAgent": {"enabled": true}
      }
    }
  }
}
```

**1.7 Добавить binding:**
```json
{
  "telegram": {
    "bindings": [
      {"peer": "1833934429", "agent": "medhelp-agent"}
    ]
  }
}
```

**1.8 Проверить allowFrom (1833934429 уже есть в списке).**
**1.9 Обновить USER.md — добавить пометку о binding medhelp-agent.**

## Этап 4 — Интеграция medcheck-agent (0.5 дня)

### Цель: настроить sessions_send и проверить контракт

### Действия

**4.1 Убедиться, что agentToAgent включён в конфиге (этап 1).**

**4.2 Прописать в AGENTS.md medhelp-agent алгоритм:**
1. Получить вопрос от Марии
2. Сформулировать ответ (черновик)
3. Извлечь факты для верификации
4. Отправить sessions_send → medcheck-agent с claims и sources
5. Получить JSON-ответ с вердиктом
6. Если ✅ — выдать ответ с цитатами и вердиктом
7. Если 🔄 — выдать обе позиции
8. Если ❓ — выдать международные источники с дисклеймером
9. Если ❌ — исправить ответ и перепроверить
10. Если таймаут — сообщить Марии

**4.3 Проверить изоляцию:** medcheck-agent не должен отвечать на внешние вызовы.

### Контрольные точки этапа 4

- [ ] AgentToAgent включён
- [ ] Алгоритм верификации прописан
- [ ] Тест: sessions_send → JSON-ответ
- [ ] Тест: изоляция (внешний запрос → отказ)
