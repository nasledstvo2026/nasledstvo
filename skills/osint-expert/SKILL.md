# Skill: osint-expert

## Описание
Специалист по OSINT-аналитике — геополитические риски, СВО, мобилизация, санкции, экономика РФ. Проект osint на сайте nasledstvo2026.github.io/nasledstvo/osint.html.

## Файлы проекта
- `projects/osint/` — папка проекта (breq.md, srs.md, test-cases.md, plan.md)

- `assets/osint-bg.jpg` — фоновое изображение

  - `osint-raw-data.json` — сырые данные Фазы A

  - `osint-log.md` — лог операций (все фазы)

## Фазная архитектура A/B/C (3 независимые cron-задачи)

### Фаза A — Сбор данных (таймаут 180 сек)
**Расписание:** 05:30, 13:30, 19:30 МСК
**Инструменты:** web_fetch, web_search, memory: read/write

**Промпт для Фазы A:**
```
Фаза A: сбор данных для OSINT-дайджеста.

1. **Обнули лог:** Запиши в memory/osint/osint-log.md только заголовок и "[YYYY-MM-DD HH:MM] Фаза A: старт"

2. **Опроси источники** через web_fetch:
   - ISW Russian Offensive Campaign Assessment (web_fetch)
   - The Guardian — Ukraine/Russia section
   - Al Jazeera — Russia/Ukraine section
   - pravo.gov.ru — свежие указы (проверить наличие новых документов о мобилизации)

3. **Поищи через SearXNG** (web_search) по ключевым запросам:
   - 'СВО мобилизация 2026'
   - 'Россия НАТО война'
   - 'специальная военная операция завершение'
   - 'Украина контрнаступление 2026'
   - 'санкции РФ экономика 2026'

4. **Фильтрация:** Исключи явно пропагандистские источники:
   - Анонимные Telegram-каналы без регистрации СМИ
   - Сомнительные сайты (домены < 30 дней, без редакции)
   - Источники без верифицированной истории
   Доверенные: ISW, The Guardian, Al Jazeera, Meduza, Коммерсантъ, РБК, официальные сайты госорганов (kremlin.ru, government.ru, pravo.gov.ru)

5. **Сохрани результат** в memory/osint/osint-raw-data.json.
   Формат JSON: массив объектов {source, date, fact, relevance}
   relevance: "svo" | "mobilization" | "nato" | "economy"

6. **Запиши в лог** memory/osint/osint-log.md:
   - "[YYYY-MM-DD HH:MM] Фаза A: ISW — успех/ошибка"
   - "[YYYY-MM-DD HH:MM] Фаза A: Guardian — успех/ошибка"
   - "[YYYY-MM-DD HH:MM] Фаза A: Al Jazeera — успех/ошибка"
   - "[YYYY-MM-DD HH:MM] Фаза A: SearXNG — найдено N результатов"
   - "[YYYY-MM-DD HH:MM] Фаза A: сохранено N фактов в raw-data"
```

### Фаза B — Анализ + генерация HTML (таймаут 300 сек)
**Расписание:** 05:50, 13:50, 19:50 МСК
**Инструменты:** read, write, memory: read/write

**Промпт для Фазы B:**
```
Фаза B: анализ и генерация OSINT-дайджеста.

1. **Прочитай** memory/osint/osint-raw-data.json

2. **Фильтрация источников (TC-02):**
   - Разреши только факты из доверенных источников: ISW, The Guardian, Al Jazeera, Meduza, Коммерсантъ, РБК, pravo.gov.ru, kremlin.ru, government.ru
   - Отсеки: анонимные Telegram-каналы, сомнительные сайты, источники без СМИ-регистрации
   - Факты с пометкой "(мнение)" — использовать с указанием, что это мнение, а не факт

3. **Дедупликация (TC-09):**
   - Прочитай memory/osint/osint-seen.md
   - Сравни связку источник + дата + утверждение — если совпадает, исключи
   - Новые факты — добавь в osint-seen.md (таблица: | источник | дата | утверждение |)

4. **Логирование (TC-12):**
   - Запиши в memory/osint/osint-log.md:
     - "[время] Фаза B: прочитано N фактов из raw-data"
     - "[время] Фаза B: отфильтровано M пропагандистских"
     - "[время] Фаза B: найдено K новых (без дублей)"
     - "[время] Фаза B: HTML сгенерирован"

5. **Анализ по 3 вопросам:**
   - 1. Вероятность завершения СВО (включая заморозку)
   - 2. Вероятность войны России с Европой/НАТО
   - 3. Вероятность второй волны мобилизации в РФ

   Для каждого вопроса:
   - Собери подтверждающие факты (≥ 2 на оценку)
   - Если факты противоречивы (TC-05): укажи обе стороны, пометь «данные противоречивы», оценка — только Средняя (🟡)
   - Выставь оценку по шкале: 🔴 Высокая (>70%), 🟡 Средняя (40-70%), 🟢 Низкая (<40%)

6. **Сформируй osint.html** со структурой:
   - Заголовок: "OSINT-дайджест — ДД.ММ.ГГГГ" + подзаголовок с датами
   - 3 раздела (по 3 вопросам), каждый с цветовой меткой вероятности
   - Раздел «Сводка ключевых событий» (дайджест без оценки)
   - Ссылка «← На главную» внизу
   - Подключи theme.css
   - Формат каждой карточки:
     ```html
     <div class="item">
       <div class="title">
         <span class="probability prob-xxx">Вероятность</span>&ensp;
         Название события
       </div>
       <div class="meta">Источник1, Источник2 · ДД.ММ.ГГГГ</div>
       <div class="body"><p>Текст с атрибуцией</p></div>
     </div>
     ```
```

3. **Meduza / The Moscow Times** — внутренняя политика РФ

## Разделы дайджеста (3 обязательных + 1 дополнительный)
1. **СВО** — динамика фронта, переговоры, завершение
2. **Война с Европой/НАТО** — угрозы, провокации, союзники
3. **Мобилизация** — открытая / скрытая, реестр, социология
4. **Сводка ключевых событий** — краткий дайджест без оценок

## Система оценок вероятности
- `prob-high` 🔴 — высокая (>70% подтверждающих фактов)
- `prob-mid` 🟡 — средняя (40-70%, или противоречивые данные)
- `prob-low` 🟢 — низкая (<40% подтверждающих фактов)
- При противоречивых данных — только Средняя (🟡)

## Принципы
1. Только факты из проверенных источников. Без домыслов.
2. Каждое утверждение — с источником + датой (связка source + date обязательна).
3. Вероятность — всегда с обоснованием и минимум 2 фактами на оценку.
4. Пропагандистские и анонимные источники — отсекать на этапе Фазы B.
5. Лог (osint-log.md) — обнуляется при старте Фазы A, заполняется всеми фазами.
6. Дедупликация — по связке источник + дата + утверждение (osint-seen.md).
7. При сбоях — failureAlert после 2 ошибок → уведомление Кириллу, cooldown 1 час.
