# SKILL.md — Вердикт юриста для katrin-agent

## Назначение
Два режима работы:
1. **Аудит документов** — проверка шаблонов, договоров, заявок, созданных katrin-agent
2. **Верификация нормы** — проверка юридических фактов (работает/не работает, точная информация/нет)

## ⚠️ Жёсткое правило
Любой ответ Катрин по юридическим вопросам **обязательно** включает верификацию. Исключений нет.

Приоритет верификации (по убыванию):
1. **consultant.ru** — приоритетный источник (через web_fetch, прямое цитирование текста НПА)
2. **Официальные НПА** — pravo.gov.ru, publication.pravo.gov.ru
3. **Профильные ресурсы** — gov-zakupki.ru, garant.ru (только если consultant.ru недоступен)

Категорически запрещено:
- Полагаться на один источник
- Использовать новостные агрегаторы как единственный источник
- Давать ответ без цитирования/ссылки на конкретный пункт НПА

---

## Режим 1. Аудит документов (как было)

### Процесс
1. **После создания документа** katrin-agent передаёт черновик auditor-agent.
2. **auditor-agent проверяет:**
   - Корректность ссылок на НПА (номер, дата, редакция)
   - Соответствие статусу закупки (активна / завершена / отменена)
   - Арифметику (НМЦК, обеспечение заявки, НДС)
   - Обязательные разделы (существенные условия)
   - Потенциальные риски (жалобы ФАС, неоднозначные формулировки)
   - Актуальность законов (через consultant.ru)

### Вердикт
```
🔍 Заключение auditor-agent:
• Риски: [HIGH / MEDIUM / LOW]
• Замечания: [список, если есть]
• Источники проверены: [список ссылок]
• Статус: [одобрено / доработка / не одобрено]
```

3. Если статус «доработка» или «не одобрено» — katrin-agent **обязан** исправить и отправить на повторную проверку.

## Режим 2. Верификация нормативного факта (новое)

### Когда запускается
Когда Катрин задаёт любой вопрос типа:
- «Точная ли информация?»
- «Работает ли X с Y-даты?»
- «Проверь, было ли продление...»
- «Что сейчас действует по ...» (любой запрос, где требуется верификация НПА)

### Процесс верификации

**Шаг 1. Идентифицировать НПА и норму**
- Какой закон/постановление? Какая статья/пункт?
- Какая дата события интересует?

**Шаг 2. Проверить consultant.ru**
- Загрузить текст НПА через web_fetch с consultant.ru
- Найти конкретный пункт/статью
- Выписать: последняя редакция, дата изменения, дата вступления в силу

**Шаг 3. Проверить альтернативный источник (если consultant не показал полностью)**
- pravo.gov.ru, gov-zakupki.ru, publication.pravo.gov.ru
- Сверить формулировки

**Шаг 4. Сформулировать вердикт**
```
🔍 Заключение auditor-agent:
• НПА: [номер, дата, редакция]
• Норма: [пункт, статья]
• Статус: [действует / истёк / изменён / не найден]
• Даты: [период действия]
• Ссылки: [consultant.ru / pravo.gov.ru]
```

### Формат ответа в чат Катрин
Верификация **обязательно** включается в конец ответа:
```
[основной ответ на вопрос]

🔍 auditor-agent:
• НПА: ПП РФ № 1875 (ред. от 23.06.2026)
• Норма: п. 4 — СТ-1 для медизделий (поз. 362-378, 383-388, 390-399, 433)
• Статус: ❌ истёк (срок до 30.06.2026)
• Ссылки: consultant.ru/...
```

## Недопустимые ситуации (можно редактировать, добавляя опыт)
- ❌ Ответ без верификации — если не удалось проверить, сказать «не могу верифицировать»
- ❌ Единственный источник — всегда минимум 2
- ❌ Источник без цитаты — ссылка + конкретный текст/пункт
- ❌ Интерпретация без прямой ссылки на текст НПА
