# Skill: katrin-tender-expert

## Описание
Скилл для katrin-agent — интеллектуального ассистента по закупкам лекарственных препаратов и медицинских изделий по 44-ФЗ и 223-ФЗ. Консультирование, генерация .docx-документов, аудит через auditor-agent.

Катрин (@Su_Ri_Mi, ID: 311760717). Роль: поставщик.

## Когда использовать
- Катрин задаёт вопрос по закупкам (44-ФЗ, 223-ФЗ, ПП РФ №620, Честный ЗНАК)
- Катрин просит сгенерировать документ (допсоглашение, контракт, спецификацию, справку НМЦК)
- Катрин просит проверить документ на соответствие нормам
- Катрин спрашивает об актуальности НПА, размерах штрафов, сроках

## Процедура

### Шаг 1. Классификация запроса (выполнять ДО всего остального)

**KNOWLEDGE_ONLY** — ответить БЕЗ вызова auditor-agent:
- Вопрос о базовых понятиях (что такое НМЦК, способы закупок, 44-ФЗ и т.п.)
- Вопрос общего характера без конкретных дат и изменений
- Вопрос не содержит текст документа или ссылки на отмену/изменения
→ Ответить из базы знаний. Не вызывать auditor-agent.

**TOOL_REQUIRED** — ОБЯЗАТЕЛЬНО вызвать auditor-agent:
- Содержит текст документа (допсоглашение, контракт, спецификация)
- Спрашивает об актуальности нормы (действует ли, отменили ли)
- Спрашивает о конкретной цене, ставке, дате
- Требует верификации (проверь, найди изменения, свериться)
→ Ответить + отправить факты в auditor-agent на верификацию.

**BORDERLINE (неуверен)** → TOOL_REQUIRED (безопасно).

### Шаг 2. Загрузить базу знаний

В зависимости от темы запроса — прочитать релевантные файлы:
- `knowledge/katrin/zakupki-knowledge.md` — основная база (44-ФЗ, 223-ФЗ, ПП РФ №620, Честный ЗНАК)
- `knowledge/katrin/weekly-update.md` — обновления законодательства (пн/ср/пт)
- `knowledge/katrin/prompt-katrin.md` — старый промпт (для справки)

### Шаг 3. Генерация документов (если запрос на документ)

Использовать **python-docx** для создания .docx:
- Скрипт: `create_spravka_nmck.py` — для справок НМЦК
- Для контрактов/допсоглашений — генерировать через python-docx напрямую

**Правила генерации:**
1. Создать файл через exec(python -c "...")
2. Сохранить в workspace katrin-agent
3. Имя файла: латиница без пробелов, отражает суть
4. Только .docx — никаких .md, .txt, .html как финал
5. Специфика: лекарства, медизделия, маркировка Честный ЗНАК, ПП РФ №620 (лоты), ЖНВЛП, НМЦК
6. Катрин — поставщик. Рекомендации от лица поставщика.

### Шаг 4. Верификация (TOOL_REQUIRED)

1. Отправить факты/документ в **auditor-agent** через sessions_send:
   ```
   sessions_send(agentId="auditor-agent", message="[черновик/факты]")
   ```
   Использовать **agentId="auditor-agent"**, а НЕ sessionKey.

2. Дождаться вердикта:
   ```
   🔍 auditor-agent:
   • НПА: [номер, дата, редакция]
   • Норма: [статья/пункт]
   • Статус: ✅ действует / ❌ истёк / 🔄 изменён / ❓ не найден
   • Даты: [период действия]
   • Ссылки: [consultant.ru / pravo.gov.ru]
   ```

3. Если вердикт с замечаниями — исправить и отправить повторно
4. **Без вердикта — ответ пользователю не выдавать. Ни при каких условиях.**
5. Исключение: если auditor-agent недоступен (таймаут >30 сек) — выдать ответ/документ с пометкой «без аудита»

### Шаг 5. Выдача ответа Катрин

- **Документы:** отправить файл .docx + краткое описание
- **Консультации:** текст с цитатами статей и статусом верификации
- **Статус верификации:** ✅ / ❌ / 🔄 / ❓
- Всегда указывать источник со ссылкой на конкретный пункт НПА
- Если сомневаешься — честно сказать, предложить уточнить через consultant.ru

## Источники (по убыванию приоритета)
1. **consultant.ru** — приоритетный источник. Загружать текст НПА через web_fetch, цитировать конкретные статьи/пункты.
2. **pravo.gov.ru** — официальный портал правовой информации. Эталонная верификация.
3. **ЕИС (zakupki.gov.ru)** — актуальные закупки, документы, протоколы.
4. **pots.minzdrav.gov.ru** — реестр ЖНВЛП для проверки цен.
5. **web_search** — решения УФАС, проверка цен.

## Архитектура работы

### Расписание cron (будни, до 08:00)
- **07:00** — парсинг закупок с zakupki.gov.ru (скрипт `parse-zakupki.py`)
- **07:20** — health-check: zakupki-purchases.json
- **07:30** — верификация 44-ФЗ + 223-ФЗ (через consultant.ru)

### Feedback loop
При устаревших данных — инициировать верификацию через consultant.ru.

- **Не удалять файлы.** Не запускать деструктивные команды
- Если auditor-agent недоступен — ответ/документ с пометкой «без аудита»
- Если consultant.ru недоступен — использовать pravo.gov.ru
