# social-verifier-protocol — Протокол юридической верификации

## Назначение
Стандартизированный протокол верификации юридических фактов (статьи НПА, размеры выплат, статусы норм, категории граждан) для мультиагентной архитектуры.

## Применение
- **social-agent → social-verifier:** отправка фактов на верификацию перед выдачей ответа пользователю
- **Перспектива:** katrin-agent → auditor-agent, advocate → advocate-verifier

---

## Формат запроса (от агента-консультанта к верификатору)

Агент отправляет через sessions_send факты строго в таком формате:

```
ВЕРИФИКАЦИЯ
Факт 1: [статья/НПА/сумма] — [что утверждается]
Факт 2: ...
---
Источники утверждения: [откуда агент взял эти данные, если есть]
```

**Правила:**
- Каждый факт с новой строки, с префиксом `Факт N:`
- Одна сессия — один запрос

## Процедура верификации (выполняется верификатором)

### Приоритет источников (строгий, не нарушать)
1. garant.ru
2. docs.cntd.ru
3. consultant.ru
4. sfr.gov.ru
5. nalog.gov.ru
6. Региональные сайты

### Типы фактов
- НПА → garant.ru → docs.cntd.ru → consultant.ru
- Выплата → sfr.gov.ru → garant.ru
- Категория → garant.ru → consultant.ru

### Статусы
| Статус | Условие |
|--------|---------|
| ✅ | Текст источника совпадает с утверждением |
| ❌ | Текст источника противоречит утверждению |
| 🔄 | НПА в новой редакции |
| ❓ | Источники недоступны |

## Формат вердикта (строгий)

```
🔍 social-verifier:
• НПА: [номер, дата, редакция]
• Норма: [статья/пункт/часть]
• Статус: ✅ / ❌ / 🔄 / ❓
• Источник: [ссылка]
• Комментарий: [только при ❌ или 🔄]
```

Если несколько фактов — перечислить последовательно.

---

## Действия консультанта с вердиктом

- ✅ — выдать ответ с пометкой "✅ проверено"
- ❌ — не выдавать. Вернуться к поиску. Если не нашёл — ответ с пометкой "требует уточнения"
- 🔄 — выдать с актуальной редакцией. Обновить knowledge/social/
- ❓ — выдать с пометкой "без аудита"

Записать результат в knowledge/social/verification-cache.md.

---

## Обновление базы знаний

При ❌ или 🔄:
1. Исправить соответствующий файл в knowledge/social/
2. Записать в weekly-update.md

## Автообновление

После каждого успешного вердикта:
- Если данные устарели (🔄) — agent обновляет knowledge/social/
- Если кеш верификации растёт >50 записей — очищать записи старше 90 дней
