# interview-doc-fill

Заполнение knowledge-баз и документов через Q&A-интервью. Адаптация interview-style-doc-building от David Ondrej.

## Когда использовать

- Нужно заполнить knowledge-базу (knowledge/katrin/, knowledge/lena/, knowledge/danil/)
- Создать USER.md для нового пользователя
- Задокументировать регламент или процесс
- Кирилл говорит «давай запишем что я знаю по теме Х»

## Процедура

1. **Создать файл** со скелетом (заголовок, секции с плейсхолдерами).
2. **Задать ОДИН вопрос.** Конкретный, открытый.
3. **Дождаться ответа.**
4. **Патчить файл** ответом в соответствующую секцию.
5. **Повторить** — следующий вопрос.
6. Пока файл не заполнен или пока Кирилл не скажет «хватит».

## Жёсткие правила

- **Один вопрос за раз.**
- **Патч, не перезапись.** После скелета — только edit.
- **Ответ → патч → следующий вопрос.**
- **Списки — неупорядоченные.** Не нумеровать без явного запроса.
- **Вопросы на раскрытие, не подтверждение.**
- **Конкретное > абстрактное.**

## Формат вопроса

Один. Конкретный. Открытый. С указанием куда пойдёт ответ.
