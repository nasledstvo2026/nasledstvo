# research-prompt-builder

Методология написания исследовательского промпта для поисковых задач. Адаптация research-prompt от David Ondrej.

## Когда использовать

- Нужно собрать информацию по новой теме (новости, законодательство, конкуренты)
- Кирилл просит «проверь что нового по X»
- Подготовка к созданию knowledge-базы для нового пользователя
- Поисковое задание для Лены (новости, РЖД), Данила (вклады), Катрин (44-ФЗ)

## Формат

Один параграф. Никаких заголовков и маркированных списков в теле промпта.

## Структура

1. **Объяснение для «нулевого контекста»** — 1-2 предложения: что за проект, почему это важно, какая ситуация
2. **Один главный вопрос**, который исследование должно ответить
3. **3-6 подвопросов** с номерами (1, 2, 3…) внутри параграфа
4. **Ограничения**: что включать, что исключать
5. **Источники**: приоритет первичных источников (официальные документы, сайты госорганов), форумы/соцсети — слабый сигнал
6. **Gap-round**: перед финалом — самокритика, поиск пробелов и противоречий, повторный проход
7. **Формат вывода**: на каждый пункт — ссылка + утверждение + строка «почему это важно»

## Процедура

1. Вытянуть контекст: даты, имена, известные факты, для чего результат
2. Сформулировать один главный вопрос
3. Разбить на 3-6 подвопросов
4. Добавить ограничения и требования к источникам
5. Сжать в один параграф
6. Выполнить: использовать web_search/web_fetch, собрать результат в markdown-файл

## Пример

> [Проект — мониторинг выплат по вкладам СССР 1991 года для граждан РФ. Сейчас собираем информацию по Сбербанку.] Исследовать актуальный порядок компенсаций по вкладам СССР в Сбербанке на 2026 год, чтобы обновить базу знаний аналитика Данила. Найти: (1) текущий размер компенсации по годам рождения вкладчика; (2) перечень необходимых документов; (3) сроки рассмотрения; (4) изменения за 2025-2026 гг. Исключить информацию по страховым выплатам и похоронным пособиям — отдельная тема. Приоритет: официальный сайт Сбербанка и ПП РФ. Если источники противоречат — разделить факты и предположения. Перед финалом проверить найденное на пропуски. По каждому пункту: ссылка + суть + почему это важно. Результат сохранить в markdown-файл.
