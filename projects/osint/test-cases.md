### TC-01: Сбор данных из источников

| | |
|---|---|
| **ID** | TC-01 |
| **Название** | Сбор данных из всех категорий источников |
| **Требования** | FR-1, раздел 5 БТ |
| **Приоритет** | 🟡 Высокий |
| **Метод проверки** | **Лог:** открыть `memory/osint/osint-log.md` после сборки и проверить записи по каждому источнику |

**Шаги:**
1. Запустить cron-задачу сбора данных
2. Открыть лог сборки `memory/osint/osint-log.md`
3. Проверить наличие записи по каждому источнику:
   - ISW (web_fetch) — статус успеха/неудачи
   - The Guardian (web_fetch) — статус успеха/неудачи

   - Pravo.gov.ru (web_fetch) — статус успеха/неудачи
   - SearXNG (поиск по ключевым запросам) — количество результатов
4. Подсчитать количество успешно опрошенных источников

**Ожидаемый результат:** ≥ 80% источников успешно опрошены.

**Критерий прохождения:** В логе присутствуют записи об опросе каждого источника с результатом.

### TC-03: Синтез фактов и предварительная оценка

| | |
|---|---|
| **ID** | TC-03 |
| **Название** | Формирование предварительной оценки по 3 вопросам |
| **Требования** | FR-2, FR-5, раздел 13.3 (Этап 2) SRS |
| **Приоритет** | 🟡 Высокий |
| **Метод проверки** | **Тестовые данные + лог:** передать эталонный набор фактов, проверить что на выходе 3 оценки, каждая ≥ 2 фактами |

3. Для каждого из 3 вопросов проверить:

   - Атрибуцию каждого факта (источник + дата)

**Ожидаемый результат:** Три оценки, каждая с ≥ 2 фактами. Каждый факт имеет атрибуцию.

**Критерий прохождения:** Все 3 вопроса имеют оценку. Каждая оценка содержит ≥ 2 подтверждающих фактов с атрибуцией.

### TC-04: Верификация фактов

| | |
|---|---|
| **ID** | TC-04 |
| **Название** | Перекрёстная проверка фактов агентом-верификатором |
| **Требования** | FR-3, раздел 13.3 (Этап 3) SRS |
| **Приоритет** | 🔴 Критический |
| **Метод проверки** | **Тестовые данные:** скармливаем заранее подготовленный набор фактов с известными статусами, сравниваем ответ верификатора с эталоном |

**Шаги:**
1. Подготовить 10 тестовых фактов с известными статусами:
   - 4 факта, подтверждённых в ≥ 2 источниках (ISW + Guardian)
   - 3 факта, подтверждённых в 1 источнике
   - 2 факта с явным опровержением в независимом источнике
   - 1 факт без возможности верификации
2. Передать список агенту-верификатору
3. Запустить перекрёстную проверку
4. Сверить результат с эталоном

**Ожидаемый результат:** 
- Факты с ≥ 2 источниками → статус «подтверждён» (4 из 4)
- Факты с 1 источником → «недостаточно подтверждён» (3 из 3)
- Факты с опровержением → исключены (2 из 2)
- Без верификации → «недостаточно подтверждён» (1 из 1)

**Критерий прохождения:** Совпадение ответа верификатора с эталоном ≥ 8 из 10 фактов.
