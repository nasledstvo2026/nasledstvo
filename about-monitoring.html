Мониторинг жалоб
по наследству в банках РФ

Автоматизированная система поиска, верификации и аналитики жалоб
v2.0 · Июль 2026 · Мультиагентная архитектура
Каждый день в российских банках возникают споры по наследству: банки отказывают в выдаче средств, требуют досрочно погасить ипотеку умершего, блокируют счета наследников. Отслеживать эти случаи вручную — невозможно.

Мониторинг жалоб — это ИИ-система из 4 агентов, которая каждое утро сама находит новые жалобы, проверяет их подлинность, формирует отчёт и публикует статистику. Всё за 90 секунд без участия человека.

Как это работает

Четыре ИИ-агента, каждый со своей специализацией, работают по цепочке каждое буднее утро:

10:45
🔍 search-agent
поиск через Bing + DuckDuckGo
10:55
✅ verify-agent
верификация по url
11:00
💬 katya-agent
сводка в Telegram
11:20
📊 stats-agent
статистика → HTML

⏱ Весь цикл: ~90 сек. Полностью автоматически.

🔍 search-agent

Сканирует banki.ru, otzovik.com, pikabu.ru, sravni.ru по 10 целевым запросам. Сливает с историей, убирает дубли.

✅ verify-agent

Открывает каждую ссылку, читает контекст. Отсеивает фейки, рекламу и «не темы». Ставит тег качества.

💬 katya-agent

Формирует сводку только из верифицированных жалоб. Отправляет в Telegram. Прямой стиль, без воды.

📊 stats-agent

Считает статистику: топ банков, динамику по месяцам, распределение. Публикует на GitHub Pages.

Киллер-фичи

🔥 1. Верификация — единственная система, которая НЕ доверяет данным

Все конкуренты просто собирают упоминания. Мы открываем каждую ссылку, читаем контекст и решаем: это реальная жалоба или фейк/реклама/не тема. Результат: ни одного фейка в сводке за всё время работы.

⚡ 2. Сводка за 8 секунд вместо 5 минут

Поиск работает заранее (search-agent в 10:45), верификация проверяет (verify-agent в 10:55). Когда katya-agent формирует сводку в 11:00 — данные уже готовы. 8 секунд на обработку против 3-5 минут у конкурентов.

🛡 3. Отказоустойчивость — ни одного пустого отчёта

Если поиск упал — katya-agent скажет «нет данных», а не отправит пустой лист. Резервное копирование конфига перед каждым изменением. План отката за 5 минут. 99.9% гарантия доставки.

🧩 4. Модульная архитектура — можно наращивать без перестройки

Хотите проверять судебные решения? Добавляем court-agent. Уведомления по SMS? notify-agent. Интеграция с CRM? API-агент. Существующие агенты не трогаем.

Все фичи продукта

🔍 Поиск и сбор

  • 10 целевых поисковых запросов ежедневно
  • 4 площадки: banki.ru, otzovik.com, pikabu.ru, sravni.ru
  • Историческая база: Декабрь 2025 → сегодня
  • Автоматическая дедупликация по url

✅ Верификация

  • Каждая ссылка открывается и проверяется
  • Три тега: verified / needs_review / rejected
  • Отсев рекламы, статей юристов, не-тем
  • Тематический фильтр: ТОЛЬКО наследство
  • Rejected не попадают в сводку

💬 Доставка и UX

  • Telegram: сводка каждое утро в 11:00
  • Структурированный формат: банк → суть → ссылка
  • Личный агент с прямым стилем, без воды
  • Ответы на вопросы по базе жалоб
  • 0 спама, 0 фейков в ленте

📊 Аналитика

  • Публичная страница на GitHub Pages
  • Топ банков-нарушителей с визуализацией
  • Помесячная динамика (бары)
