# AI DJ — Архитектура модуля

## 1. Структура директорий

```
/home/user1/.openclaw/workspace/aidj/
├── ARCHITECTURE.md          # этот файл
├── aidj-handler.py          # хэндлер команды /dj (сессия, запрос файлов, ответ)
├── aidj-engine.py           # движок сведения (изолированный, вызывается как subprocess)
├── config.py                # конфигурация (пути, порты, BPM-диапазон)
├── static/                  # веб-сервер для раздачи аудио
│   └── .gitkeep
├── output/                  # временные файлы миксов
│   └── .gitkeep
└── aidj.html                # лендинг с БТ (публикуется на GitHub Pages)
```

## 2. Компоненты

### 2.1. Хэндлер `/dj` (aidj-handler.py)

- Регистрируется как команда OpenClaw
- При вводе `/dj` → запрашивает Track A (ждёт аудиофайл)
- После получения Track A → запрашивает Track B
- После получения Track B → запускает aidj-engine.py в фоновом subprocess
- Отвечает пользователю: «Микс готов: <ссылка>»

**Защита основного процесса:**
- Движок — отдельный процесс (subprocess), не thread.
- Таймаут 120 секунд на обработку.
- Если движок упал — возвращается ошибка, основной бот не страдает.

### 2.2. Движок сведения (aidj-engine.py)

Запускается: `python3 aidj-engine.py track_a.mp3 track_b.mp3 --output mix_001.mp3`

Алгоритм:
1. Определить BPM обоих треков (через ffmpeg + aubio)
2. Выровнять темп: растянуть/сжать более медленный трек до BPM быстрого (ffmpeg atempo)
3. Выполнить crossfade 12-15 секунд (ffmpeg acrossfade)
4. Сохранить результат в `output/` и скопировать в `static/` для веб-доступа

**Зависимости:** ffmpeg (уже установлен), python3 (встроенный subprocess)

### 2.3. Веб-сервер для аудио

- Python3 http.server на порту 8765 (systemd unit или screen)
- Раздаёт файлы из `aidj/static/`
- Адрес: `http://<vps-ip>:8765/mix_001.mp3`

### 2.4. Лендинг (aidj.html)

- Публикуется на GitHub Pages как https://nasledstvo2026.github.io/nasledstvo/aidj.html
- Содержит БТ и положения инициативы
- Позже: встроенный аудиоплеер

## 3. Изоляция от новостных дайджестов

| Аспект | Решение |
|--------|---------|
| Процесс | Движок — отдельный subprocess, не влияет на main |
| Память | subprocess умирает после выполнения |
| Время | Таймаут 120с, если завис — просто ошибка |
| Команда | Только `/dj`, не пересекается с cron |
| Файлы | Вся музыка — в `aidj/`, не трогает memory/ и отчёты |

## 4. BPM-детекция и сведение

### Варианты:
1. **aubio** (pip install aubio) — точный BPM, C-библиотека, быстрый
2. **ffmpeg + astats** — встроенный, менее точный
3. **librosa** — тяжёлый, много зависимостей

**Выбор (MVP):** `aubio` для BPM + `ffmpeg` для обработки.

### Crossfade:
- ffmpeg filter `acrossfade=d=15` — плавный переход 15 секунд
- Предварительное выравнивание BPM через `atempo`
