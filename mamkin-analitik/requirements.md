# Проект «Мамкин аналитик» — Документ требований

> Версия: 1.0 (MVP)
> Дата: 08.07.2026
> Статус: требования утверждены
> Пользователи: Владелец продукта / Бизнес-аналитик (равноценные, работают по одному)

---

## 1. Описание проблемной области

**Пользователи сервиса:**
- Владелец продукта (предметник, отвечает на вопросы)
- Бизнес-аналитик (равноценный пользователь, работает самостоятельно)

**Текущая ситуация (боль):**
- БТ делают долго
- Используют разные шаблоны — нет единого формата
- Часто формальный подход, неглубоко
- Не формулируют проблемную область, не выходят на цель
- Подготовка БТ занимает много времени или о ней забывают
- Новые инициативы появляются часто — БТ нужен на каждую и быстро

---

## 2. Описание цели

**Конечный результат работы сервиса:**
- Готовый документ БТ в формате **DOCX**
- Публикация идентичной версии на веб-странице

**Принцип работы:**
- Сервис сам опрашивает пользователя (агентный подход)
- Вопросы динамически меняются в зависимости от ответов
- Агент погружается в суть предмета, адаптирует следующие вопросы

---

## 3. Критерии достижения цели

**Признаки успеха:**
- Время подготовки БТ сократилось
- БТ стали глубже, проработаннее
- Каждая новая инициатива получает БТ (не забывают)
- Пользователю легко и понятно проходить опрос
- Хороший документ БТ на выходе говорит сам за себя

**Метрики:** не отслеживаются (качество БТ — экспертный критерий)

---

## 4. Контекст и ограничения

**Инфраструктура (что есть):**
- VPS (vm-f13581, Linux)
- GitHub Pages для статики
- DeepSeek API (LLM)
- Cloudflare Tunnel (HTTPS)

**Ограничения сети:**
- ❌ Прямой доступ к VPS по HTTP/IP заблокирован в корпоративной сети
- 🤔 Cloudflare Tunnel (trycloudflare.com) — HTTPS, **проверяется**
- ℹ️ Если Cloudflare тоже заблокирован — нужен иной подход

**Объём БТ:**
- Зависит от контекста инициативы
- Формулировки без лишней художественности, по делу

## 5. Шаблон БТ (неменяемый) — утверждён для MVP

> ⚠️ Шаблон фиксируется на этапе разработки сервиса и не подлежит изменению после ввода в эксплуатацию. Все вопросы агента строятся на основе этой структуры.

### Структура шаблона БТ (MVP):

| № | Раздел | Описание |
|---|--------|----------|
| 1 | **Название инициативы** | Что за проект, краткое название |
| 2 | **Проблемная область** | Что сейчас не так, какая боль у бизнеса/пользователей |
| 3 | **Цель** | Что хотим получить, измеримый результат |
| 4 | **Заинтересованные стороны** | Кто участники, кто влияет на проект, чьи интересы затронуты |
| 5 | **Функциональные требования** | Что система/решение должно делать |
| 6 | **Нефункциональные требования** | Производительность, безопасность, доступность, совместимость |
| 7 | **Ограничения** | Бюджет, сроки, технологии, корпоративные политики, регуляторика |
| 8 | **Критерии приёмки** | Как поймём, что задача сделана и принята |
| 9 | **Риски** | Что может пойти не так, что может помешать достижению цели |

### Примечания к шаблону:
- Агент последовательно проходит по разделам, задавая уточняющие вопросы
- Вопросы динамические — зависят от ответов пользователя
- Разделы заполняются нелинейно: агент может возвращаться к разделам для уточнения
- Все разделы обязательны к заполнению (даже ответом «не применимо»)

## 6. Архитектура решения

### Канал взаимодействия:
- Пользователь общается с агентом через **Telegram** (стандартный канал)
- Агент задаёт вопросы, анализирует ответы, динамически адаптирует следующие вопросы

### Генерация документа:
- Agent (OpenClaw / DeepSeek) формирует БТ на основе утверждённого шаблона
- DOCX-генерация выполняется на VPS (python-docx)

### Публикация:
- DOCX-файл и веб-версия публикуются на **GitHub Pages**
- Пользователь получает ссылку в Telegram

```
[Пользователь (Telegram)] ↔ [OpenClaw / DeepSeek (диалог, сбор требований)]
                                                              ↓
                                          DOCX + веб-версия → GitHub Pages
                                                              ↓
                                             Ссылка в Telegram пользователю
```

### Компоненты:
- **Канал:** Telegram (бот → пользователь)
- **Agent:** OpenClaw + DeepSeek API — опрос по шаблону, генерация БТ
- **Документ:** python-docx (VPS) для генерации .docx
- **Публикация:** GitHub Pages (DOCX + статичная веб-страница с содержанием)

### Формат финального документа:
- **DOCX** — скачивание
- **Веб-версия** — идентичное содержание на GitHub Pages

### Хранение:
- Требуется уточнить (история сессий, черновики)
