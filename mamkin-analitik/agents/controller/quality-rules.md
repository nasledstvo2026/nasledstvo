# Quality Rules — Правила проверки качества БТ

## Критерии достаточной глубины (per-dimension)

Оценка производится по каждому измерению глубины для каждого блока.

### Блок 1 — Контекст и проблема

| Измерение | Критерий | Оценка 0-10 |
|-----------|----------|-------------|
| **Контекст** | Описана инициатива: название, суть, область | Есть название и суть → 7+, только «проект» → 0-3 |
| **Специфика проблемы** | Конкретные примеры, описание последствий | Есть примеры и цифры → 7+, общие фразы → 0-3 |
| **Root Cause Analysis** | Анализ причин проблемы | Есть «из-за того что X → Y» → 7+, нет причин → 0-3 |
| **Последствия бездействия** | Конкретные цифры потерь | Есть цифры потерь → 7+, общие слова → 0-3 |

### Блок 2 — Цель и измерение

| Измерение | Критерий | Оценка 0-10 |
|-----------|----------|-------------|
| **Измеримость цели** | Бизнес-цель с KPI и целевыми значениями | Есть цифры и сроки → 7+, только «улучшить» → 0-3 |
| **Метрики успеха** | 2+ измеримых критерия | Каждый с метрикой+значением → 7+, без цифр → 0-3 |
| **Временные рамки** | Срок достижения цели | Есть квартал/дата → 7+, нет срока → 0-3 |

### Блок 3 — Стейкхолдеры и влияние

| Измерение | Критерий | Оценка 0-10 |
|-----------|----------|-------------|
| **Идентификация групп** | Конкретные группы/роли | Есть отделы, должности → 7+, «все» → 0-3 |
| **Интересы** | Интересы каждой группы | Для каждой описан интерес → 7+, нет → 0-3 |
| **Влияние** | Степень влияния на проект | Кто решает, утверждает → 7+, нет → 0-3 |

### Блок 4 — Решение

| Измерение | Критерий | Оценка 0-10 |
|-----------|----------|-------------|
| **Функциональные требования** | 3+ конкретные функции | Есть глагол+суть+условие → 7+, «удобно» → 0-3 |
| **Приоритеты** | Приоритет для каждой функции | MVP/Release 2/NTH указаны → 7+, нет → 0-3 |
| **Пользовательские сценарии** | Happy path + ошибки | Описан flow → 7+, только слова → 0-3 |
| **Граничные случаи** | 2+ edge case | Есть «что если» → 7+, нет → 0-3 |

### Блок 5 — Требования к качеству и ограничения

| Измерение | Критерий | Оценка 0-10 |
|-----------|----------|-------------|
| **Качественные характеристики** | Конкретные SLA | Есть цифры и метрики → 7+, «быстрая» → 0-3 |
| **Ограничения** | Бюджет, сроки, технологии | Конкретные цифры → 7+, общее → 0-3 |

### Блок 6 — Приёмка

| Измерение | Критерий | Оценка 0-10 |
|-----------|----------|-------------|
| **Критерии для каждой функции** | Каждая функция имеет критерий | Есть Given-When-Then → 7+, нет → 0-3 |
| **Формат сценариев** | Для 2+ критичных функций | Формат соблюдён → 7+, частично → 0-3 |

### Блок 7 — Риски и митигации

| Измерение | Критерий | Оценка 0-10 |
|-----------|----------|-------------|
| **Количество рисков** | 3+ риска | 3+ → 7+, 1-2 → 3-6, 0 → 0 |
| **Оценка рисков** | Вероятность и влияние | Оба указаны → 7+, частично → 3-6 |
| **Митигация** | Конкретные шаги | Есть действия → 7+, «следить» → 0-3 |

## Vague Language — Размытые формулировки (shallow detection)

### Паттерны, указывающие на поверхностный ответ

| Категория | Примеры | Severity |
|-----------|---------|----------|
| **Оценочные прилагательные** | качественный, удобный, современный, эффективный, хороший | high |
| **Общие существительные** | система, продукт, решение, платформа | medium |
| **Неопределённые величины** | много, мало, быстро, долго, часто | high |
| **Пустые глаголы** | улучшить, оптимизировать, автоматизировать | high |
| **Канцелярит** | осуществлять взаимодействие, производить улучшение | medium |
| **Фразы-паразиты** | в целом, в принципе, в основном, как правило | low |

### Маркеры глубины

| Маркер | Проверка |
|--------|----------|
| ✅ Есть цифры | Ответ содержит числа |
| ✅ Есть примеры | Ответ содержит «например», «в частности» |
| ✅ Есть причины | Ответ содержит «потому что», «из-за», «в результате» |
| ✅ Есть имена/роли | Ответ содержит должности, названия отделов |
| ✅ Есть функции (для блок 4) | Ответ содержит конкретные действия системы |
| ❌ Нет цифр → shallow | Для блоков 1, 2, 5, 7 |
| ❌ Нет анализа причин → shallow | Для блока 1 |
| ❌ Нет метрик → shallow | Для блока 2 |
| ❌ Нет стейкхолдеров → shallow | Для блока 3 |
| ❌ Нет функций → shallow | Для блока 4 |

## Scoring per depth dimension

| Балл | Уровень | Описание |
|------|---------|----------|
| 9-10 | Глубоко | Конкретные факты, цифры, анализ, примеры |
| 7-8 | Достаточно | Есть конкретика, но не хватает деталей |
| 5-6 | Поверхностно | Затронуто, но без цифр/фактов |
| 3-4 | Слабо | Упомянуто формально |
| 0-2 | Отсутствует | Не раскрыто |

## Пороги

- **PASS**: все измерения ≥ 6
- **NEEDS_IMPROVEMENT**: 1-2 измерения < 6
- **FAIL**: ≥ 3 измерений < 6

---

## Правила возврата
- Если блок не соответствует критериям → вернуть на доработку
- Возвращать с указанием конкретного измерения и его балла
- Максимум 3 итерации на блок
- После 3 итерации — принять как есть с пометкой
